# Thesis implementation status

This project implements a Python, theorem-aware version of GSNH-style multivariate decision trees inspired by the journal paper on tractable explanations of multivariate decision trees.

## Safe thesis statement

“The goal is not only to beat CART accuracy. The goal is to obtain competitive accuracy with smaller trees and shorter abductive explanations, while keeping theorem-certified results separate from auxiliary and empirical extensions.”

## What is implemented

- Greedy decision-tree training with interpretable split languages.
- Threshold literals and multivariate predicates for 1D, Horn, AntiHorn, ConjUI, Square2CNF, and Affine/XOR-style auxiliary paths.
- AXp-style explanation checks with backend metadata describing which satisfiability backend was used.
- Benchmark scripts that separate theorem-certified rows from auxiliary and empirical rows.

## Relationship to the paper idea

The implementation follows the paper idea of restricting path constraints to language families with tractable satisfiability checks. It is an implementation inspired by the theory, not a formally verified extraction of the theory.

## Theorem-certified families in this Python implementation

The theorem-certified benchmark table is restricted to rows where theorem mode was actually used and the backend certificate is accepted:

- 1D threshold paths, via the structural Horn-style threshold backend.
- Horn paths, via `structural_horn`.
- AntiHorn paths, via `structural_antihorn`.
- Square2CNF paths, only when the backend is `two_sat` and the path certificate is `2cnf`.

These rows must also have `theorem_mode_used=True` and `theorem_certified=True`.

## Auxiliary or empirical families

- ConjUI is useful empirically but is not included in the theorem-certified benchmark table unless a path receives one of the accepted certificates above.
- Affine uses an auxiliary GF(2)-style backend in Python. It is not theorem-certified in the Python benchmark because there is no verified Python GF(2) certificate checker in this implementation.
- BestPN is empirical. It chooses the locally best split among the families that are actually searched at a node. It is not a global oracle and is not theorem-certified.

## What BestPN searches

BestPN searches a local candidate pool containing Horn, AntiHorn, and ConjUI candidates. Affine is excluded by default and is searched only if `allow_affine_in_bestpn=True`. Square2CNF is not part of BestPN in this implementation.

## What the benchmark shows

The benchmark measures empirical accuracy, tree size, training time, and explanation lengths. Accuracy or size improvements are empirical observations, not formal guarantees. The theorem-certified table is deliberately narrower than the full results table.

## What is not claimed

- The Python implementation is not fully formally verified.
- The Python implementation is not extracted from Coq.
- Greedy training is not globally optimal.
- BestPN is not theorem-certified.
- Affine paths are not theorem-certified in Python.
- Mixed affine+numeric empirical paths do not have certified subset-minimal AXps.

## Known limitations and future work

- Mixed affine+numeric empirical paths preserve sound path satisfiability checks, but the deletion-based AXp procedure can be over-conservative for subset-minimality because it does not solve a combined numeric plus GF(2) minimality problem.
- This limitation does not affect theorem-certified benchmark rows because those rows exclude Affine and fallback backends.
- The 3D Affine loop was audited: the `i` loop generated by `iters_i = ni // step` visits the same threshold sequence as `range(step, ni + 1, step)`, so the suspected asymmetry is not a bug.
- Future work could add a verified GF(2) certificate checker, a mixed numeric+GF(2) minimality solver, or a separately certified BestPN variant.
